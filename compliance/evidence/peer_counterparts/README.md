# External Peer Counterpart Catalog

These manifests define the independent counterpart classes accepted for Tier 4 interop bundles.

A counterpart definition is not evidence by itself. Tier 4 promotion requires preserved external artifacts,
peer identity, runtime/container provenance, scenario outcomes, a preserved reproduction path, and a declared
independence attestation class of `independent-external`.

## Counterpart families

- browser / SPA / RP harnesses
- native and device client harnesses
- assertion-capable client harnesses
- registration-management harnesses
- resource-server and gateway harnesses
- sender-constrained DPoP and mTLS harnesses
- advanced PAR/JAR/RAR client harnesses
- operator CLI harnesses
- Uvicorn / Hypercorn / Tigrcorn runner harnesses
